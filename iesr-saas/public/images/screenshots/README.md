# System screenshots for the landing page

Drop PNG (or JPG) screenshots here and they appear automatically on the landing
page — no code change needed. The page checks which files exist at build/request
time and falls back gracefully (branded IESR panel) when a file is missing, so
nothing ever shows as a broken image.

## Admin dashboard (Administrators role tab)

| Save as | What to capture |
|---|---|
| `admin-dashboard.png` | The admin **Analytics & insights** dashboard (full page) |

## Active-cohort cards (one register screenshot per class)

File name = the class code with every non-letter/digit replaced by `-`, `.png`:

| Class | Save as |
|---|---|
| CEEMAY2025R | `CEEMAY2025R.png` |
| CEESEP2024A | `CEESEP2024A.png` |
| CEESEP2024B | `CEESEP2024B.png` |
| DEEMAY2024A | `DEEMAY2024A.png` |
| DEEMAY2024B | `DEEMAY2024B.png` |
| DEEMAY2025R | `DEEMAY2025R.png` |
| DEESEP2023R | `DEESEP2023R.png` |
| CDAC/L5/EE/SEP2025 | `CDAC-L5-EE-SEP2025.png` |
| CDAC/L6/EE/SEP2025 | `CDAC-L6-EE-SEP2025.png` |

When you add a **new class** in Admin, it automatically appears as a cohort card
(branded IESR panel). To give it a register preview, just save a screenshot named
after its code using the same rule above.

Tip: keep files reasonably small (≤ ~400 KB each) so the page stays fast.
